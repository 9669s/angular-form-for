Templates and Overrides

The formFor library offers several options for HTML templates. These options are separated into modules that are included with each formFor release (as of version 1.3.x).

Bootstrap Templates

formFor includes a set of HTML templates that are compatible with the Twitter Bootstrap framework. To use these templates:

  1. Include form-for.bootstrap-templates.js
  2. Require the formFor.bootstrapTemplates module as a dependency.

Default Templates

formFor also provides a set of HTML templates and CSS styles that define a custom look-and-feel. To use these templates:

  1. Include form-for.default-templates.js and (optionally) form-for.css
  2. Require the formFor.defaultTemplates module as a dependency.

Custom Templates

You can also choose to implement your own template HTML using Angular's $templateCache.

For instance to override the markup used by the <text-field> directive you could use either of the following approaches:

Define a simple inline template

$templateCache.put('form-for/templates/text-field.html', '
Your custom HTML here
');

Load an external template

$http.get('your/custom/template.html').success(function(html) { $templateCache.put('form-for/templates/text-field.html', html); });

The key thing to note is the template URL format used by formFor components.

Override styles

The formFor distribution includes optional, default stylesheets. You can choose to provide your own stylesheets instead simply by not including the styles bundled with formFor.

For a complete list of configurable styles, check out the formFor templates.
