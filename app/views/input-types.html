Input Types

formFor supports the following field types:

Used for text input. Specify a type attribute to configure additional behavior (like password). Specify a multiline attribute to enable multiline text input.
Drop-down/select menu for enumerations. Requires an options parameter to specify the set of valid values. Add an allow-blank attribute to signify optional fields.
Checkbox for Boolean values.
Radio button for enumerations. This type of component is automatically grouped by its attribute attribute to ensure that only one value is selected at a time.
(Optional) submit button directive. This button automatically binds to formFor and disables itself during submission. For a complete list of attributes for the above directives, refer to the API Reference page.

For a complete list of attributes for the above directives check out the documentation.

Creating a custom field

You can also create a custom field directive as follows:

formFor fields have a lot of capabilities, but the only requirements are:

  • It must register with the via the registerFormField method.
    • The simplest way of doing this is to use FieldHelper.manageFieldRegistration as shown above.
  • The registerFormField method will return an object containing several keys. Your view should use these keys as follows:
    • Read/write to the bindable attribute.
    • Observe the disabled attribute.
    • Display the error message (if one exists).
